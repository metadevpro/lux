Select

Usage samples

Multiple/single Select editor. Allows to select data for a collection value.

Main features:

  • Separation between key (programatic value used) and label (shown on user interface).

Sample

    
obj = {
  countries: [  'Spain', 'Italy', 'Costa Rica', 'Philippines' ]
}
    
  
    
<lux-select id="countries"
               [(ngModel)]="obj.countries"
               [unique]="unique"
               [multiple]="multiple"
               [disabled]="disabled"
               [placeholder]="placeholder">
</lux-select>
Configuration:

Current value is: 
{{ obj.countries | json }}

Documentation

The LuxSelect component implements ControlValueAccessor, therefore binding using ngModel and name is supported.

Properties

  • idUnique Id in the DOM to ensure accesibility (autogenerated by default).
  • disabled If disabled the component is not editable. (Default: false)
  • value Current value.
  • multipleSupport for multiple selected values. (Default: true)
  • uniqueShould selected values be unique? (Default: true)
  • placeholderText to show when adding new items. (Default: 'Add new')
  • validateItemOptional validation function. It must implement the signature: validateItem: (item: string) => string. Returns error if any does not pass validation as string or null if no error. (Default: (_) => null )

Events

  • valueChange Fires whenever the value changes.
